## 🔧 **预期的问题和解决方案**

### **常见启动问题:**

1. **容器启动失败**
   - 检查端口占用: `ss -tlnp | grep [端口]`
   - 检查容器日志: `podman logs [容器名]`

2. **API服务无响应**
   - 可能需要更长启动时间
   - 检查环境变量配置 (.env文件)
   - 验证API密钥设置

3. **数据库连接问题**
   - 检查PostgreSQL容器状态
   - 验证数据库初始化脚本
   - 确认网络连接

4. **n8n工作流执行失败**
   - 检查微服务API端点可达性
   - 验证工作流配置
   - 检查认证设置
